# 봇 관리

기관 관리자는 Workhub에서 사용할 봇을 등록하고 관리합니다.

## 봇이란?

봇은 자동화된 작업을 수행하는 프로그램입니다. 채널에 알림을 보내거나, 외부 서비스와 연동하는 데 사용합니다.

## 봇 목록

관리 > **봇 관리** 메뉴에서 등록된 봇 목록을 확인합니다.

## 봇 등록

1. **"+ 봇 추가"** 버튼을 클릭합니다.
2. 다음 정보를 입력합니다:
   - 봇 이름
   - 설명
   - 프로필 이미지 (선택)
3. **"등록"** 버튼을 클릭합니다.
4. 등록 후 **API 키**가 발급됩니다.

> **주의**: API 키는 등록 시 한 번만 표시됩니다. 안전한 곳에 저장해 주세요.

## 봇 설정

### API 키 재발급

1. 봇 상세에서 **"키 재발급"** 버튼을 클릭합니다.
2. 기존 키는 즉시 무효화되며, 새 키가 발급됩니다.

### 봇 비활성화

1. 봇 상세에서 **"비활성화"** 버튼을 클릭합니다.
2. 비활성화된 봇은 API 호출이 차단됩니다.

## 시스템/플랫폼 봇

Workhub에 기본 내장된 시스템 봇(검색 봇, 태스크 봇, 알림 봇)과 플랫폼 봇(스탠드업, 투표/설문, 취합, 태스크 알림)은 **시스템 관리자만** 수정/삭제/키 관리가 가능합니다.

일반 기관 관리자는 시스템/플랫폼 봇을 **목록에서 확인**할 수 있지만, 편집이나 삭제 버튼은 표시되지 않습니다.

| 봇 유형 | 기관 관리자 | 시스템 관리자 |
|---------|-----------|-------------|
| 사용자 등록 봇 | 조회/수정/삭제 가능 | 모든 권한 |
| 시스템/플랫폼 봇 | 조회만 가능 | 모든 권한 |

## 봇 API 사용

봇은 REST API를 통해 다음 작업을 수행할 수 있습니다:

- 채널에 메시지 전송
- DM 전송
- 채널 목록 조회
- 사용자 정보 조회

API 요청 시 헤더에 API 키를 포함합니다:

```
Authorization: Bearer {API_KEY}
```
