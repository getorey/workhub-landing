# 프로젝트

프로젝트는 Workhub의 핵심 업무 단위입니다. 프로젝트 안에 **토픽**(주제별 논의)을 만들고, 각 토픽에서 메시지와 태스크를 관리합니다.

## 프로젝트 목록

사이드바 > **프로젝트**를 클릭하면 참여 중인 프로젝트 목록이 표시됩니다.

## 프로젝트 생성

1. 프로젝트 목록 화면에서 **"+ 프로젝트 만들기"** 버튼을 클릭합니다.
2. 프로젝트 이름과 설명을 입력합니다.
3. **토픽 정책**을 선택합니다:
   - `필수`: 모든 메시지가 토픽 단위로 관리됩니다.
   - `선택`: 토픽 없이도 사용 가능합니다.

## 프로젝트 상세

프로젝트를 클릭하면 다음 정보를 확인할 수 있습니다:

- **토픽 목록**: 프로젝트 내 토픽들
- **구성원**: 프로젝트 참여자 목록
- **프로젝트 설정**: 이름, 설명 수정

## 토픽

토픽은 프로젝트 내의 주제별 논의 공간입니다.

### 토픽 생성

1. 프로젝트 상세 화면에서 **"+ 토픽 만들기"** 버튼을 클릭합니다.
2. 토픽 이름과 설명을 입력합니다.

### 토픽 상태

| 상태 | 설명 |
|------|------|
| 진행중 | 현재 활동 중인 토픽 |
| 완료 | 목적을 달성한 토픽 |
| 보관 | 더 이상 사용하지 않는 토픽 |

### 토픽 상세

토픽을 클릭하면 두 개의 탭이 표시됩니다:

- **메시지** 탭: 토픽 내 대화, 쓰레드, 리액션
- **태스크** 탭: 칸반 보드 또는 간트 차트 뷰

## 프로젝트 구성원

### 역할

| 역할 | 권한 |
|------|------|
| admin | 프로젝트 설정, 구성원 관리, 보관 |
| member | 토픽 생성, 메시지, 태스크 |
| guest | 읽기 전용 |

### 구성원 초대

프로젝트 관리자(admin)가 프로젝트 상세 > 구성원 섹션에서 사용자를 초대합니다.
