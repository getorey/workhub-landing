# 봇 사용 가이드

Workhub에는 업무 자동화를 도와주는 **4가지 봇**이 내장되어 있습니다. 봇은 채널에서 슬래시 명령어(`/`)로 실행하거나, DM을 통해 자동으로 알림을 받을 수 있습니다.

## 봇 목록

| 봇 | 주요 기능 | 실행 방법 |
|----|----------|----------|
| [데일리 스탠드업 봇](#데일리-스탠드업-봇) | 매일 아침 스탠드업 자동화 | 자동 (DM) |
| [태스크 알림 봇](#태스크-알림-봇) | 마감일 알림 & 독촉 | 자동 (DM) |
| [투표/설문 봇](#투표-설문-봇) | 채널 투표 & DM 설문 | 슬래시 명령어 |
| [취합 봇](#취합-봇) | 팀원 의견/자료 취합 | 슬래시 명령어 |

## 데일리 스탠드업 봇

매일 아침 팀원에게 DM으로 스탠드업 질문을 보내고, 답변을 모아 채널에 리포트를 게시합니다.

### 동작 흐름

```
09:00  DM으로 스탠드업 질문 발송
       ↓ 팀원이 DM으로 답변
09:30  미응답자에게 리마인더 발송
10:00  채널에 스탠드업 리포트 게시
```

### 받게 되는 질문

봇이 DM으로 3가지 질문을 순서대로 보냅니다:

1. **어제 한 일** — 어제 완료한 업무를 알려주세요
2. **오늘 할 일** — 오늘 진행할 업무를 알려주세요
3. **블로커** — 업무 진행에 방해되는 것이 있나요?

### 답변 방법

1. 봇이 보낸 DM에 **텍스트로 답변**하면 됩니다
2. 질문이 순서대로 오므로 하나씩 답변합니다
3. 블로커가 없으면 "없음"이라고 답변합니다
4. 모든 답변이 완료되면 봇이 확인 메시지를 보냅니다

### 스탠드업 리포트

10:00에 채널에 자동으로 게시되는 리포트에는 다음이 포함됩니다:

- 📊 응답률 (응답/전체 인원)
- 각 팀원의 어제 한 일, 오늘 할 일
- 🚨 블로커가 있는 경우 별도 표시

::: tip
스탠드업 시간이나 대상 채널은 관리자가 설정합니다. 변경이 필요하면 기관관리자에게 문의하세요.
:::

## 태스크 알림 봇

담당하고 있는 태스크의 마감일이 가까워지면 DM으로 알림을 보내줍니다.

### 알림 시점

| 시점 | 알림 내용 |
|------|----------|
| **D-3** | 마감 3일 전 — 사전 알림 |
| **D-1** | 마감 1일 전 — 긴급 알림 |
| **D-Day** | 마감일 당일 |
| **D+1** | 마감 1일 초과 — 지연 경고 |
| **D+3** | 마감 3일 초과 — 에스컬레이션 |

### DM 알림 예시

```
⏰ 태스크 마감 알림

📋 "API 문서 작성" — 마감까지 D-1
📁 프로젝트: Workhub v2.0

아래 내용으로 답변해주세요:
  • "완료" — 태스크를 완료 처리
  • "진행중" — 진행중으로 상태 변경
  • "연장" — 마감 연장 요청
```

### 답변 방법

봇이 보낸 알림 DM에 아래 키워드로 답변하면 자동으로 처리됩니다:

| 답변 | 동작 |
|------|------|
| **완료** | 태스크 상태를 "완료"로 변경 |
| **진행중** | 태스크 상태를 "진행중"으로 변경 |
| **연장** | 마감 연장 요청 (관리자에게 알림) |

### 일일 태스크 리포트

매일 09:30에 지정된 채널에 자동으로 리포트가 게시됩니다:

- 마감 임박 태스크 목록
- 지연된 태스크 목록
- 각 담당자별 현황

::: warning
태스크에 마감일을 설정해야 알림을 받을 수 있습니다. 칸반 보드에서 태스크를 편집하여 마감일을 지정하세요.
:::

## 투표/설문 봇

채널에서 실시간 투표를 진행하거나, DM 기반의 설문조사를 실행할 수 있습니다.

### 투표 만들기

채널 메시지 입력창에서 슬래시 명령어를 입력합니다:

```
/투표 제목 | 항목1 | 항목2 | 항목3
```

#### 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--익명` | 익명 투표 (누가 투표했는지 비공개) | `/투표 점심 메뉴 | 한식 | 중식 --익명` |
| `--복수` | 복수 선택 허용 | `/투표 회식 날짜 | 월 | 화 | 수 --복수` |
| `--시간 30분` | 제한 시간 설정 (기본: 60분) | `/투표 긴급 안건 | 찬성 | 반대 --시간 10분` |

#### 예시

```
/투표 이번 주 회식 장소 | 고깃집 | 횟집 | 이탈리안 | 중식당 --복수 --시간 120분
```

### 투표 참여 방법

1. 봇이 채널에 투표 카드를 게시합니다
2. **번호를 입력**하여 투표합니다 (예: `1`, `2`, `3`)
3. 복수 선택이 허용된 경우 쉼표로 구분합니다 (예: `1,3`)
4. 5표마다 중간 결과가 자동으로 표시됩니다
5. 제한 시간이 지나면 최종 결과가 게시됩니다

### 설문조사 만들기

DM 기반의 다중 질문 설문을 실행합니다:

```
/설문 분기 만족도 조사
```

#### 옵션

| 옵션 | 설명 |
|------|------|
| `--익명` | 익명 설문 |

### 설문 참여 방법

1. `/설문` 명령을 실행하면 채널 구성원에게 DM이 발송됩니다
2. 봇 DM에서 질문에 하나씩 답변합니다
3. 텍스트 답변 또는 점수(1~5) 평가를 합니다
4. 모든 응답이 수집되면 채널에 결과 리포트가 게시됩니다

### 설문 결과 리포트

- 텍스트 응답: 주요 키워드 분석
- 점수 응답: 평균 점수 계산
- 응답률 표시

## 취합 봇

팀원들에게 DM으로 자료나 의견을 요청하고, 마감일에 자동으로 결과를 모아 채널에 게시합니다.

### 취합 요청 만들기

```
/취합 제목 | 마감: 날짜 | 항목: 항목1, 항목2
```

#### 예시

```
/취합 주간 업무 보고 | 마감: 2026-04-11 | 항목: 이번주 성과, 다음주 계획, 이슈사항(선택)
```

- **제목**: 취합 주제
- **마감**: 마감일 (YYYY-MM-DD 형식)
- **항목**: 수집할 항목 목록 (쉼표 구분)
- 항목 뒤에 `(선택)`을 붙이면 선택 항목이 됩니다

### 응답 방법

1. 봇이 채널 구성원에게 DM으로 취합 요청을 보냅니다
2. 각 항목에 대해 **순서대로 텍스트로 답변**합니다
3. 답변 완료 전에 **검토 및 수정**할 수 있습니다
4. 최종 제출 후 요청자에게 알림이 갑니다

### 진행 상황 확인

취합을 요청한 사람은 진행 상황을 확인할 수 있습니다:

```
/취합현황
```

- 응답 완료/미응답 인원 확인
- 실시간 응답률 표시

### 리마인더

- 매일 오전 9시에 미제출자에게 자동으로 리마인더가 발송됩니다
- 마감일에 자동으로 결과 보고서가 채널에 게시됩니다

### 결과 보고서

마감 시 채널에 자동으로 게시되는 보고서에 포함되는 내용:

- 📊 응답률 (제출/전체)
- 각 팀원의 항목별 응답 내용
- 미응답자 목록

::: tip
선택 항목은 응답하지 않아도 제출이 가능합니다. 필수 항목은 반드시 답변해야 합니다.
:::

## 슬래시 명령어 요약

| 명령어 | 봇 | 설명 |
|--------|-----|------|
| `/투표 제목 \| 항목1 \| 항목2` | 투표/설문 봇 | 채널 투표 생성 |
| `/설문 제목` | 투표/설문 봇 | DM 설문조사 생성 |
| `/취합 제목 \| 마감: 날짜 \| 항목: ...` | 취합 봇 | 자료 취합 요청 |
| `/취합현황` | 취합 봇 | 취합 진행 상황 확인 |

::: info
슬래시 명령어는 채널 메시지 입력창에서 `/`를 입력하면 자동완성 목록이 표시됩니다.
데일리 스탠드업 봇과 태스크 알림 봇은 자동으로 동작하므로 별도 명령어가 필요 없습니다.
:::

## 봇에게 답변하는 팁

1. **DM에서 답변** — 봇이 보낸 DM 대화방에서 바로 텍스트를 입력하면 됩니다
2. **간결하게** — 긴 문장보다 핵심 내용을 간결하게 작성하세요
3. **순서대로** — 여러 질문이 오는 경우 하나씩 순서대로 답변합니다
4. **키워드 답변** — 태스크 알림 봇은 "완료", "진행중", "연장" 등 키워드로 답변합니다

::: warning 주의사항
- 봇은 **텍스트 메시지만** 인식합니다 (이미지, 파일은 인식하지 않음)
- 봇의 응답 데이터는 메모리에 저장되므로, **서버 재시작 시 진행 중인 세션이 초기화**될 수 있습니다
- 봇 관련 문제가 발생하면 기관관리자에게 문의하세요
:::
